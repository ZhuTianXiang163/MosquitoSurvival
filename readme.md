# VR 灭蚊游戏 README

## 项目简介

本项目是一款基于 Unity XR Interaction Toolkit 搭建的 VR 灭蚊游戏。玩家在室外场景中移动、观察、采集材料、使用道具，并通过电蚊拍击杀不断刷新的蚊子。游戏包含基础战斗、蚊子生成与攻击、金币奖励、血量管理、背包与商店、室外采集、特殊彩蛋以及 UI 界面反馈等模块。

整体玩法围绕“探索场景、采集资源、合成或购买道具、击杀蚊子并维持生存”展开。玩家可以通过击杀蚊子获得金币，也可以在室外区域采集花和草，用于合成恢复、防护或清场类道具。随着游戏进行，玩家需要合理使用背包、商店和场景中的特殊交互物体，应对蚊群攻击和复仇机制带来的压力。

## 1. 基础玩法与蚊子战斗系统

### 1.1 电蚊拍交互与命中判定

游戏中使用网球拍模型作为电蚊拍的视觉表现，并在拍面位置添加独立的 `Swatter_HitZone` 作为命中检测区域。该区域通过 Trigger Collider 检测是否接触蚊子，同时结合拍面移动速度判断是否属于有效挥拍。

这种设计可以避免玩家只是轻微碰到蚊子就被判定为击中，使击杀行为更接近真实挥拍操作。

### 1.2 蚊子目标与移动行为

蚊子模型使用 Anything World 资源，并添加 `MosquitoTarget`、Collider 和 Rigidbody，使其能够作为可被击打的目标参与游戏逻辑。

蚊子通过脚本实现围绕玩家飞行、随机扰动和靠近玩家等行为。移动方式既能体现对玩家的追踪感，也避免蚊子完全按照机械直线移动，从而增强游戏过程中的动态变化。

### 1.3 蚊子刷新系统

蚊子的生成由 `MosquitoSpawner` 统一管理。场景中设置多个 Spawn Zone，用于控制蚊子主要从室外、草丛、水边等区域刷新。

刷新系统会限制场上同时存在的蚊子数量。当蚊子被击杀后，系统会继续补充新的蚊子，保证游戏过程持续进行。

### 1.4 金币奖励系统

金币由 `GameManager` 统一维护。玩家每击杀一只蚊子会获得固定金币奖励，目前设定为 `80` 金币。

金币变化通过事件向外部模块暴露，便于 UI 系统实时读取并更新金币显示，也方便商店、背包等系统与金币逻辑进行联动。

### 1.5 击中特效与奖励反馈

当蚊子被击中后，系统会在击打位置附近播放电火花特效，并显示短暂的 `+80` 奖励文本。奖励文本会在停留一段时间后淡出，用于增强玩家击中目标后的反馈感。

### 1.6 血量与死亡逻辑

玩家血量由 `GameManager` 统一管理。蚊子靠近玩家并满足叮咬条件后，会触发扣血逻辑。

当玩家血量归零后，游戏进入死亡状态。此时系统会停止继续刷新蚊子，停止蚊子的叮咬逻辑，并禁止玩家继续击杀蚊子。血量变化和死亡事件同样会向外部开放，便于 UI 制作血条、死亡界面和 Game Over 反馈。

### 1.7 蚊群复仇机制

游戏实现了“蚊群复仇”机制。当玩家在短时间内连续击杀多只蚊子时，会触发复仇事件，一次性生成一批更具攻击性的蚊子。

该功能由 `SwarmRevengeManager` 监听击杀事件，并调用 `MosquitoSpawner` 生成复仇蚊群。复仇蚊子的移动速度、感知距离和叮咬频率更高，会形成类似“捅了马蜂窝”的紧张效果。

## 2. VR 操作与交互控制

本项目基于 Unity XR Interaction Toolkit 和 XR Device Simulator 实现桌面端 VR 模拟操作。为了提升灭蚊游戏中的操作流畅度，对默认 XR Device Simulator 的控制方式进行了优化，使玩家可以更自然地在移动、观察、控制左右手柄和抓取物体之间切换。

优化后的核心目标是：玩家抓取电蚊拍后，不需要一直按住抓取键，也不需要停止移动才能挥拍。电蚊拍会持续保持在抓取它的手上，直到玩家再次按下抓取键主动放下。

### 2.1 基础操作方式

| 输入              | 功能          |
| --------------- | ----------- |
| `W / A / S / D` | 控制玩家水平移动    |
| `Q / E`         | 控制玩家下降 / 上升 |
| 鼠标移动            | 控制第一人称视角    |
| 鼠标左键 + 鼠标移动     | 控制左手柄方向     |
| 鼠标右键 + 鼠标移动     | 控制右手柄方向     |
| `G`             | 抓取 / 放下物体   |

玩家可以通过鼠标调整视角，按住鼠标左键或右键时切换为对应手柄控制。将手柄瞄准可抓取物体后，按 `G` 即可抓取；再次按 `G` 时放下。

### 2.2 控制方式优化

项目保留 Unity 自带的 `XR Device Simulator` 和 XR Interaction Toolkit 基础交互逻辑，包括左右手柄模拟、鼠标控制视角、鼠标控制手柄方向以及 `G` 键触发 Grip 抓取。

在默认控制方式基础上，项目对手柄操作进行了简化：

```text
Manipulate Left  -> Mouse Left Button
Manipulate Right -> Mouse Right Button
```

修改后，按住鼠标左键时鼠标控制左手柄，按住鼠标右键时鼠标控制右手柄，不按鼠标键时鼠标控制玩家视角。

同时，项目移除了默认 XR Device Simulator 中键盘平移手柄的相关绑定，使 `W / A / S / D / Q / E` 始终用于移动玩家本体，避免在手持物体时移动物体。新增移动脚本会根据 `Main Camera` 朝向计算移动方向，让玩家在控制视角、左手柄或右手柄时都能正常移动。

### 2.3 Toggle 抓取方式

为了让玩家抓住电蚊拍后能够持续移动和挥拍，左右手柄 Interactor 的抓取触发方式被设置为 Toggle：

```text
按一次 G：抓取物体
再按一次 G：放下物体
```

这样玩家可以先用手柄瞄准电蚊拍并按 `G` 抓取，然后继续移动、转向和挥拍，电蚊拍会保持在手上，直到再次按下 `G`。

## 3. 室外场景与采集系统

### 3.1 室外场景搭建

游戏包含可探索的室外区域。场景中摆放了花、草、植物、装饰物和特殊交互物体，为采集、战斗和探索提供空间基础。

为了方便玩家识别可采集资源，花草采集物添加了发光粒子效果。场景物体也进行了结构整理，便于后续维护和扩展。

### 3.2 花草采集系统

室外区域中设置了花和草两类采集物。玩家可以使用 XR 手柄瞄准采集物，并按下交互键进行采集。

采集成功后，玩家会获得对应材料。被采集的花草模型和粒子特效会暂时消失，并在一段时间后自动刷新，使玩家可以持续在场景中收集材料。

采集到的花和草可用于后续合成道具，例如花露水、草药和生命药剂。

### 3.3 特殊彩蛋交互

场景中还设置了特殊彩蛋交互物体，玩家与其交互后可以获得额外效果或奖励。

* **大师之剑**：交互后获得一次复活机会。玩家受到致命伤害时不会死亡，并恢复到满血状态。
* **圣诞老人**：交互后获得圣诞礼物，包括 1 瓶花露水和 1 瓶草药。

两个彩蛋交互均为一次性触发，为玩家提供探索奖励和额外生存机会。

## 4. 背包、物品与商店系统

### 4.1 背包系统

游戏实现了物品网格背包。背包初始容量为 12 格，每格独立存放一个物品，不进行堆叠。

背包支持按分类筛选物品，包括：

* 全部
* 材料
* 消耗品
* 工具

背包还支持一键整理排序，排序顺序为：

```text
花 → 草 → 草药 → 花露水 → 灭蚊喷雾 → 生命药剂
```

同类物品会按数量排列。玩家点击物品后，会弹出详情面板，显示物品名称、描述、获取途径、持有数量和售价，并可在详情面板中直接使用或出售物品。

玩家死亡后，系统会禁止继续使用物品。

### 4.2 物品系统

游戏目前包含 6 种主要物品：

| 物品   | 分类  | 用途                | 获取方式    |
| ---- | --- | ----------------- | ------- |
| 花    | 材料  | 合成花露水 / 生命药剂 / 出售 | 室外采集    |
| 草    | 材料  | 合成草药 / 生命药剂 / 出售  | 室外采集    |
| 草药   | 消耗品 | 回复 10 HP          | 合成或商店购买 |
| 花露水  | 消耗品 | 免疫蚊子伤害 10 秒       | 合成或商店购买 |
| 灭蚊喷雾 | 消耗品 | 清除场景内所有蚊子         | 商店限时购买  |
| 生命药剂 | 消耗品 | 回复全部血量            | 合成获得    |

所有物品数据由 `ItemDatabase` 统一管理，支持图标加载和运行时查询，便于背包、商店和 UI 模块统一读取物品信息。

### 4.3 商店系统

商店包含两个页面：合成与购买页面、背包扩容页面。

#### 合成与购买页面

玩家可以在商店中合成或购买部分道具：

| 道具   | 合成条件                 | 购买价格        |
| ---- | -------------------- | ----------- |
| 花露水  | 花 ×2                 | 100 金币      |
| 草药   | 草 ×2                 | 60 金币       |
| 生命药剂 | 花 ×3 + 草 ×3 + 200 金币 | 不作为普通购买项    |
| 灭蚊喷雾 | 不通过材料合成              | 180 金币，限时商品 |

其中，灭蚊喷雾为限时商品，每 3 分钟刷新库存。使用后可以清除场景内所有蚊子。

商店会实时显示玩家金币和材料数量。当条件不足时，对应按钮会变灰并提示原因；购买或合成成功时会弹出提示，失败时会通过状态栏显示消息。

#### 背包扩容页面

玩家可以消耗金币扩展背包容量：

| 扩容档位 | 增加格数 |  价格 | 可购买次数上限 |
| ---- | ---: | --: | ------: |
| 小型扩容 |   +4 | 150 |     3 次 |
| 中型扩容 |   +8 | 400 |     2 次 |
| 大型扩容 |  +16 | 900 |     1 次 |

通过扩容，背包最多可扩展到 44 格。

## 5. 游戏界面 UI 系统

UI 系统用于展示游戏状态、引导玩家理解操作，并在游戏不同阶段提供明确反馈。界面内容覆盖游戏中 HUD、启动界面、暂停界面、玩法介绍界面、死亡界面、结算界面和状态提示等部分。

### 5.1 游戏中 HUD

游戏运行时，HUD 会显示玩家当前金币数量和血量信息，例如：

```text
Coins: 0
HP: 100 / 100
```

同时，界面中包含血条，用于直观展示玩家剩余生命值。金币和血量会根据游戏中的实际变化实时更新，使玩家能够随时了解当前状态。

### 5.2 启动界面

启动界面用于游戏开始前的导航。界面提供开始游戏、设置、玩法介绍等入口，让玩家在正式进入游戏前了解基本操作和游戏目标。

### 5.3 暂停界面

暂停界面用于游戏过程中的临时中断。玩家可以通过暂停菜单继续游戏、重新开始或查看玩法介绍，便于在游戏中随时回顾说明或重置流程。

### 5.4 玩法介绍界面

玩法介绍界面整理并展示游戏的基础操作和核心目标，包括玩家移动、视角控制、左右手柄控制、抓取与放下物体等内容。

同时，该界面也介绍了主要游戏目标和道具效果，例如击杀蚊子获得金币、使用草药回血、使用花露水免疫蚊子伤害等，帮助玩家理解不同系统之间的关系。

### 5.5 死亡界面与结算界面

当玩家死亡后，系统会显示死亡提示界面，并提供重新开始等操作。

游戏结束后的结算界面用于展示本局游戏结果和玩家获得的金币等信息，使游戏流程有明确的结束反馈。

### 5.6 游戏状态提示

游戏过程中出现重要事件时，UI 会显示对应提示文字。例如：

* 获得采集物
* 获得道具
* 蚊群复仇触发
* 玩家死亡

这些提示将部分调试信息转化为玩家能够理解的界面反馈，提高游戏可玩性和信息清晰度。

## 6. 基础玩法流程

玩家进入游戏后，可以在室外场景中移动和观察环境，寻找发光的花和草进行采集。采集到的材料会进入背包，并可用于合成花露水、草药和生命药剂等道具。

在战斗过程中，玩家可以使用手柄抓取电蚊拍，通过挥拍击杀靠近的蚊子。击杀蚊子后会获得金币奖励，并触发击中特效和奖励文本反馈。金币可以用于商店购买道具或扩容背包。

玩家需要注意自身血量。蚊子靠近并叮咬玩家后会造成伤害，血量归零后进入死亡状态。玩家可以使用草药或生命药剂恢复血量，也可以使用花露水在短时间内免疫蚊子伤害，或使用灭蚊喷雾清除场景中的所有蚊子。

当玩家短时间内击杀多只蚊子时，可能触发蚊群复仇机制，生成更具攻击性的蚊群。玩家需要结合移动、挥拍、道具使用和资源管理来维持生存。

场景中还隐藏了特殊彩蛋。与大师之剑交互可以获得一次复活机会，与圣诞老人交互可以获得花露水和草药奖励。这些彩蛋为探索过程提供了额外收益，也增强了游戏的趣味性。

## 7. 模块关系概览

各模块之间通过事件、数据和玩家操作相互连接：

* `GameManager` 负责金币、血量、死亡状态等核心游戏数据。
* `MosquitoSpawner` 负责普通蚊子和复仇蚊群的生成。
* `SwarmRevengeManager` 监听击杀事件，并在条件满足时触发蚊群复仇。
* `Swatter_HitZone` 负责电蚊拍命中检测和有效挥拍判定。
* `ItemDatabase` 统一管理物品数据，供背包、商店和 UI 查询。
* 背包系统负责物品存储、筛选、整理、使用和出售。
* 商店系统负责道具购买、材料合成、限时商品和背包扩容。
* 室外采集系统为背包和商店提供花、草等基础材料。
* UI 系统负责显示金币、血量、道具说明、游戏状态提示、死亡和结算反馈。

通过这些模块的配合，游戏形成了从场景探索、资源采集、道具管理到灭蚊战斗和生存挑战的完整流程。
